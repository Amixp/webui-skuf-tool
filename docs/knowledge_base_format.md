# Рекомендуемый формат записи в базе знаний Open WebUI

## Общие требования

- Одна запись = один объект знаний (инцидент, проблема, задача, изменение).
- Метаданные хранятся в полях записи (key/value) в WebUI Knowledge.
- Текстовое описание хранится в основном поле контента/описания записи.

## Обязательные поля

Минимальный набор:

- `id` (например, `PBI000000183188`, `INC000021013376`, `TAS000013445262`)
- `service` (название услуги)

## Рекомендуемые поля

- `priority`
- `status`
- `category`
- `class`
- `created_at`
- `resolved_at`
- `closed_at`
- `coordinator`
- `assignee`
- `summary`
- `resolution`
- `last_comment`

## Пример записи

```text
id: PBI000000183188
service: Видеокомфорт
priority: Medium
status: Closed
created_at: 2023-09-01 02:21:52
summary: Потеря видеоархива у всех пользователей (INC000018102347).
resolution: Разнесение VIP IP, настройка failover.
```

## Почему так лучше для поиска

- Метадата легко фильтруется (например, `service=Видеокомфорт`, `status=Closed`).
- Текстовое описание остается свободным, можно индексировать целиком.
- Единые ключи упрощают запросы и SQL allowlist.
