# Рекомендуемый формат базы знаний

## Общие требования

- Один документ = одна запись (инцидент, проблема, задача, изменение).
- Записи разделяются строкой `---`.
- Вверху записи размещается метадата в формате `Ключ: значение`.
- После первого пустого ряда начинается основной текст (описание/решение/комментарии).

## Обязательные поля

Минимальный набор:

- `Problem ID` **или** `Номер инцидента` **или** `Номер задачи` **или** `Номер изменения`
- `Сервис` **или** `Имя услуги`

## Рекомендуемые поля

- `Приоритет`
- `Статус`
- `Категория`
- `Класс`
- `Дата создания`
- `Дата решения`
- `Дата закрытия`
- `Координатор`
- `Назн группа`
- `Назн лицо`
- `Описание`
- `Решение`
- `Последний коммент`

## Пример записи

```text
Problem ID: PBI000000183188
Сервис: Видеокомфорт
Приоритет: Medium
Статус: Closed
Дата создания: 2023-09-01 02:21:52

Описание: Потеря видеоархива у всех пользователей (INC000018102347).
Решение: Разнесение VIP IP, настройка failover.
```

## Почему так лучше для поиска

- Метадата легко фильтруется (например, `Сервис=Видеокомфорт`, `Статус=Closed`).
- Текстовое тело остается свободным, можно индексировать целиком.
- Рекомендовано хранить исходные русскоязычные ключи, а при индексации приводить к `snake_case`.
